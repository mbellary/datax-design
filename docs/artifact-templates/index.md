# Design Artifact Templates

Use these templates when adding new DataX design material. They keep artifacts
consistent without forcing every document into a heavy process.

## Templates

| Template | Use When |
|---|---|
| [Architecture Overview](architecture-overview.md) | Starting a major subsystem or capability design |
| [Sequence Diagram](sequence-diagram.md) | Explaining interaction order across actors/components |
| [Flow Chart](flow-chart.md) | Showing branching logic, lifecycle, or system flow |
| [Container Diagram](container-diagram.md) | Defining deployable/runtime units and dependencies |
| [API Contract](api-contract.md) | Capturing request/response/event contracts |
| [Data Model](data-model.md) | Defining durable entities, fields, and invariants |
| [ADR](adr.md) | Recording a consequential architecture decision |
